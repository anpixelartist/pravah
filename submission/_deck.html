The problem

Enforcement is reactive, and measured in the wrong currency

  • Police count violations and deploy by gut — but 500 helmet tickets can cause zero congestion, while one truck at a chokepoint costs thousands of vehicle-hours.
  • There is no impact heat-map and no easy way to prioritise junctions.
  • The data only shows where officers already stood — so models trained on it just tell police to keep patrolling the same corners. The data lies by omission.

The blind spot is the opportunity03
The solution

One pressure score. A map you can actually read.

  • Every violation re-expressed as hours of stuck traffic (severity × vehicle footprint).
  • One pressure score (0–100) per junction — like an air-quality index for congestion.
  • A map-first command centre on a real Bengaluru street base — tap any junction for the full story. Runs 100% offline: no tiles, no APIs.
Pressure, not ticket counts04
Glass box, by design

Every score decomposes. Every number can be challenged.

  • Pressure = how bad the parking is + evening blind spot + number of cases — shown in points.
  • Click any junction → the raw tickets behind the score, and the cost of ignoring it.
  • Plain language only — no jargon a constable can't act on. Tune the priorities and the whole city re-ranks live (the client mirrors the Python engine exactly).

From insight to action

An officer-deployment plan — with a reason on every pick

  • Set your real officer count → Pravah places them to clear the most stuck-traffic hours, with a one-line reason and an equity check across police areas.
  • We're honest: ticket-count and impact correlate 0.98, so re-ranking alone adds only ~3–5%.
  • The decisive win is structural — Pravah's picks cover the evening blind hours and phantom spots that count-based patrols never see.

Why it earns trust

Honest inputs → transparent reasoning → human decides

  • FACT / EST / AI on every figure; plain language everywhere.
  • Only the provided dataset for analysis; no external/live feeds. Demo runs fully offline.
  • Coverage-normalised features correct the enforcement bias instead of reinforcing it.
  • Human-in-the-loop — recommend, never auto-dispatch.

Under the hood

  • Pure Python engine: pipeline → features → pressure / optimise / model, wrapped by explain / validate / equity.
